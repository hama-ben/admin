## BUG 1 — Announcements: "badge_type" column not found error
═══════════════════════════════════════════════════════════════

When trying to publish an announcement, this error appears:
"Could not find the 'badge_type' column of 'announcements' 
in the schema cache"

### Fix:
First check the ACTUAL announcements table structure in Supabase:
  SELECT column_name FROM information_schema.columns 
  WHERE table_name = 'announcements';

Then do ONE of the following:

Option A — If badge_type column does not exist in Supabase:
  Add it via migration:
  ALTER TABLE announcements 
  ADD COLUMN IF NOT EXISTS badge_type text DEFAULT 'info';
  
  Then reload Supabase schema cache:
  Go to Supabase Dashboard → Settings → API → 
  click "Reload schema cache"

Option B — If the admin form is sending badge_type but the 
  database column has a different name:
  Update the form submission code to use the correct column name
  that actually exists in the table

Option C — If badge_type is not needed at all:
  Remove the Badge Type field from the announcement form entirely
  and do not send it in the insert query

Also check target_audience column — make sure it exists with 
the correct name in Supabase before inserting.

After fix: Admin should be able to publish announcements 
with no errors and they should appear in "Past Announcements".
